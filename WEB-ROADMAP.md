# 星穹飞剑 · 仙侠岛 Web 原型开发说明

## 当前定位

「仙侠岛」从 iOS 原型调整为「星穹飞剑」生态下的 Web/PWA 产品入口，优先验证：

- 仙侠岛主场景是否有探索与展示吸引力
- 角色换装、飞剑装备是否能形成生态资产感
- 修行数据、背包、好友是否能承接后续玩法扩展

## 技术栈决策

当前阶段只开发 Web/PWA，不继续投入 iOS 原生端。

| 层 | 技术 | 说明 |
|----|------|------|
| 前端框架 | React | 主应用、页面状态和交互 |
| 构建工具 | Vite | 本地开发和生产构建 |
| 语言 | TypeScript | 数据模型和资产映射 |
| 样式 | CSS | 响应式布局、场景动效和 UI 视觉 |
| PWA | vite-plugin-pwa | manifest、安装入口和缓存策略 |
| 持久化 | localStorage | 当前角色、飞剑、偏好配置 |

iOS 原型代码保留在仓库中作为历史验证资产，后续不新增 SwiftUI、SpriteKit、ActivityKit 或 Widget 相关功能，除非 Web/PWA 阶段验证完成后重新立项。

## 已完成的 Web MVP

- React + Vite + TypeScript 前端工程
- 全屏星穹云海主场景
- Q 版角色、主浮岛、子岛、传送门、云雾、飞剑环绕动效
- WASD / 方向键角色移动
- 移动端虚拟摇杆
- 六个核心模块导航：
  - 仙侠岛
  - 换装
  - 飞剑库
  - 背包
  - 好友
  - 修行
- 换装实时同步主场景角色
- 飞剑装备实时同步主场景与状态栏
- 用户配置保存到 localStorage
- 桌面和移动端响应式布局
- 生产构建通过

## 本地运行

```bash
npm install
npm run dev
```

打开：

```text
http://localhost:5173/
```

生产构建：

```bash
npm run build
```

## 下一阶段建议

1. 批量生成美术资产库：主岛精修、子岛、NPC、建筑、交互点、UI 状态件。
2. 增加 PWA 配置，让手机浏览器可以添加到主屏幕。
3. 把飞剑库、背包、好友拆成可复用数据模型，为后端或 Supabase 接入做准备。
4. 增加岛屿交互点：传送门、NPC、好友拜访。
5. 设计运动数据转化规则：跳绳/修行时长 -> 灵力 -> 飞剑养成/岛屿装饰。
